## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `.env.local` → `publish-cezari.cjs` → tfx-cli | The PAT crosses from disk → Node process env → spawnSync argv → tfx HTTP request. The `--token` flag puts the PAT on argv (visible in OS process tables briefly during publish). The dev-publish.cjs design rationale called this out; the Phase 5 replacement preserves the same posture. |
| `package.json` scripts → CI runner | The scripts run in CI exactly as locally. `check:size` reads only `dist/` (post-build); no env vars or secrets. `publish:*` is NOT run in CI (D-2). |
| GitHub Actions runner ↔ npm registry | `npm ci` downloads packages over HTTPS using `package-lock.json` integrity hashes. setup-node's npm cache stores `~/.npm` keyed by lockfile hash. |
