<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner -> repository (`release` branch, tags) | The workflow pushes commits + tags using a minted GitHub App installation token; untrusted only insofar as the workflow YAML itself is the trust root |
| Repo secrets (`APP_ID`, `APP_PRIVATE_KEY`, `TFX_PAT`) -> workflow steps | Secrets cross into step env / action inputs; must never be echoed, never land in commits, never appear in run logs |
| GitHub Actions Marketplace action (`actions/create-github-app-token`) -> the minted token | A compromised/typo'd action version could exfiltrate the App private key — mitigated by major-pinning to a maintained, official (`actions/` org) action |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Information Disclosure | `APP_PRIVATE_KEY` / `APP_ID` in `publish.yml` | mitigate | Referenced only as `${{ secrets.APP_PRIVATE_KEY }}` / `${{ secrets.APP_ID }}` passed to `create-github-app-token` inputs; never `echo`-ed, never in a `run:` step body, no `set -x`. GitHub auto-redacts secret values in logs. The action revokes the minted token at job end. |
| T-08-02 | Tampering / Supply chain | `actions/create-github-app-token` action version | mitigate | Pinned to `@v2` — a maintained major of the *official* `actions/`-org action (not a third-party fork). The executor confirms the action's `action.yml` outputs before committing. (SHA-pinning is out of scope per the project's `@v<major>` convention — REQUIREMENTS "Out of Scope".) |
| T-08-03 | Elevation of Privilege | App installation token scope | mitigate | `permission-contents: write` + `permission-pull-requests: write` only — least privilege; the token is further capped by what the App was installed with (the user installs it with exactly `contents: write` + `pull-requests: write` + implicit `metadata: read` in Plan 03). No `administration`, no `actions`, no org-level scope. |
| T-08-04 | Denial of Service (self-inflicted) | publish re-trigger loop on `release` | mitigate | `[skip ci]` (lowercase, literal) in the bot commit message — load-bearing now (an App token, unlike `GITHUB_TOKEN`, re-triggers workflows); PLUS the job-level actor-guard extended to `story-point-release-bot[bot]`; PLUS `concurrency: { group: publish-release, cancel-in-progress: false }` (queues, never overlaps). |
| T-08-05 | Tampering | `publish.yml` itself edited to weaken gates / swap the publish token | accept | Out of scope for Phase 8 (a CODEOWNERS pin is noted as a future nicety in RESEARCH §Security); the existing Phase 6/7 gates are preserved verbatim and the change is reviewed via this plan's execution + the milestone close.
