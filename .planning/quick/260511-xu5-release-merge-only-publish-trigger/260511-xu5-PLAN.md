# Quick Task 260511-xu5: Restrict Marketplace publish to merged PRs into `release`

## Goal

The Publish workflow currently fires on **any** push to `release`. Change it so the
Marketplace publish runs **only when a pull request is merged into `release`** — plus the
existing `workflow_dispatch` escape hatch for manual runs.

## Decision

- Trigger on `pull_request` `closed` against `release`; gate the job with
  `github.event.pull_request.merged == true` so non-merged closes are ignored.
- Keep `workflow_dispatch`.
- Keep `paths-ignore` so a docs-only merge to `release` doesn't republish.
- Checkout must explicitly use `ref: release` — on a `pull_request: closed` event the default
  checkout ref is the PR merge ref, not the branch tip; and for `workflow_dispatch` `release`
  is the correct target anyway.

## Tasks

### Task 1 — Rewrite the `on:` trigger and job `if:` guard in `.github/workflows/publish.yml`

- **files:** `.github/workflows/publish.yml`
- **action:**
  - Replace `on.push` (branches `[release]`) with `on.pull_request` (`types: [closed]`,
    `branches: [release]`), preserving `paths-ignore`. Keep `workflow_dispatch`.
  - Update the `publish` job `if:` to also require
    `(github.event_name == 'workflow_dispatch' || github.event.pull_request.merged == true)`
    in addition to the existing bot-actor exclusions.
  - Add `ref: release` to the `actions/checkout@v5` step's `with:`.
- **verify:** `python -c "import yaml,sys; yaml.safe_load(open('.github/workflows/publish.yml'))"` (or any YAML linter) parses clean; visually confirm the `on:`, `if:`, and checkout blocks read as intended.
- **done:** `publish.yml` triggers only on merged PRs into `release` (+ manual dispatch), checks out the `release` branch, and the rest of the steps are untouched.

## Must-haves

- `truths:`
  - Publish workflow no longer triggers on direct pushes to `release`.
  - Publish workflow still runs on a PR merged into `release`.
  - Publish workflow still runs on manual `workflow_dispatch`.
- `artifacts:`
  - `.github/workflows/publish.yml` updated.
- `key_links:`
  - `.github/workflows/publish.yml`
