**Latest version:** v1.0.7 (2026-05-04)

## Overview

The journey shipped a public Marketplace Azure DevOps work item extension that lets a team member produce a justified, reproducible Story Points value in under 30 seconds. Built integration-first: Phase 0 unblocked the build (npm versions, Marketplace publisher, atomicity decision, manifest skeleton with locked scope); Phase 1 built the pure calc engine and sentinel audit parser with zero ADO surface contact; Phase 2 deployed a minimal manifest shell that proved the highest-risk integration step (toolbar contribution + dialog open + theme inheritance) before any React UI work; Phase 3 layered the full modal UI and read path on top of the proven shell; Phase 4 wired the write path with the atomicity ordering chosen in Phase 0 plus permission and edge-case handling; Phase 5 hardened the bundle, produced Marketplace listing assets, and published to Visual Studio Marketplace under publisher `TsezariMshvenieradzeTfsAiReviewTask`. Post-publish patch sequence v1.0.1..v1.0.7 closed UX polish and the programmatic-close story (Quick task `260504-cl1`).

## Milestone Summary

### Decimal Phases
None inserted during v1.0 — milestone proceeded in linear integer order 0 → 1 → 2 → 3 → 4 → 5.

### Key Decisions

- **Apply atomicity:** comment-first → field-write (Phase 0 D-01) — successful comment + failed field is recoverable via parser pre-fill; reverse order loses provenance.
- **Manifest scope locked at `vso.work_write` only** (Phase 0) — adding scopes post-publish forces re-consent across every install.
- **React + TypeScript with `azure-devops-ui`** (Phase 0) — ADO Extension SDK is React-first; native components only available for React; consistency with ADO chrome.
- **Sentinel HTML-comment + JSON payload + `schemaVersion` for audit format** (Phase 1) — survives ADO renderer; round-trip parseable; schemaVersion enables v2 dimension expansion.
- **Pre-fill from current SP value via Phase 3 read path** + sentinel parser as test-only contract (Phase 4 D-02 STRIPPED-FALLBACK) — ADO storage strips `<!-- -->` regardless of api-version; round-trip pre-fill from sentinel deferred to v1.x reopen-pre-fill.
- **No upfront isReadOnly probe** (Phase 4 D-07 LAZY-FALLBACK-ONLY) — no reliable probe exists in dialog iframe; reactive read-only UX via FieldFailBanner is the production baseline.
- **`openCustomDialog` → `addDialog` swap** (Quick 260504-cl1, v1.0.5) — discovered post-Phase-5 that `closeDialog()` only manages the addDialog stack; matched-pair primitive choice unlocks programmatic close.
- **Publisher swap to `TsezariMshvenieradzeTfsAiReviewTask`** (Phase 5) — original `TsezariMshvenieradzeExtensions` publisher's private state was stuck server-side blocking public flip.

### Issues Resolved

- ADO sentinel comment round-trip viability — falsified at Plan 04-01 spike; D-02 fallback adopted.
- isReadOnly probe mechanism — falsified across 4 candidates at Plan 04-01 spike A3; reactive baseline locked.
- Programmatic dialog close — initially locked NO-PROGRAMMATIC-CLOSE; reversed in Quick 260504-cl1 via addDialog swap.
- Webpack multi-entry HTML output for toolbar + modal contributions.
- Theme inheritance via `applyTheme: true` + `azure-devops-ui` Surface/Page (no detection code).
- Iframe Esc keydown isolation from host (browser security model) — solved with iframe-local listener routing to closeProgrammatically.
- Cross-process FieldResolver (StoryPoints / Size) with cache + D-20 default fallback.
- Apply two-leg orchestrator atomicity with isDirty no-op skip + plain-object SDK rejection classification (Phase 4 fix-back 4).

### Issues Deferred (carry-over to v1.0.1+ / v1.1+)

- **PKG-04** Contributor non-admin explicit smoke test skipped per CONTEXT D-7 (trusted `vso.work_write` scope).
- **PKG-05** light + dark screenshots[] omitted from v1.0.0 manifest to unblock publish; deferred to v1.0.1+.
- **PKG-07** cross-process Agile + CMMI smoke (Plan 05-04) — FieldResolver code path 100% unit-tested but production smoke not run.
- **D-17 Scenarios 3/5/7** (offline / Stakeholder license / slow-3G) deferred from cezari verification; orchestrator code paths covered by 398/398 unit tests.
- **APPLY-03 production wire-format mismatch** — postComment.ts writes plain text (D-02), but parse.ts SENTINEL_RX still requires the HTML-comment sentinel. Pre-fill flow is structurally dead in production for v1-authored comments. Either widen parse.ts to accept human-readable form (option b) or strip orphaned `serialize` re-export and document as v1 known-limitation (option a).
- **closeProgrammatically defense-in-depth** — assumes addDialog stack target is still our dialog; no `is this still my dialog?` guard. spcSaving cross-component handshake is string-keyed.
- **PermissionWarnBanner structural dead code** — gated on literal `false` per spike A3 LAZY-FALLBACK-ONLY; component retained for a future probe-validated path.
- **publish:cezari script is dead** — combination of `--share-with cezari` + `public: true` manifest is rejected by Marketplace; remove or guard against `manifest.public === true`.
- **Old publisher cleanup** — `TsezariMshvenieradzeExtensions` stuck-private extension still listed; unpublish/delete is low-priority housekeeping.
